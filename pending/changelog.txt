🔧 **Claude Code アップデート**
**v2.1.119**
・`/config` の設定（theme、editor mode、verbose 等）が `~/.claude/settings.json` に保存され、プロジェクト/ローカル/ポリシーの上書き優先順位に参加するようになった
・フッターの PR バッジを github.com ではなく任意のコードレビュー URL に向ける `prUrlTemplate` 設定を追加
・起動ロゴの作業ディレクトリを非表示にする `CLAUDE_CODE_HIDE_CWD` 環境変数を追加
・`--from-pr` が GitLab マージリクエスト、Bitbucket プルリクエスト、GitHub Enterprise の PR URL を受け付けるようになった
・`--print` モードがエージェントの `tools:` および `disallowedTools:` フロントマターを尊重するようになり、インタラクティブモードの動作と一致するようになった
・`--agent <name>` が組み込みエージェントの `permissionMode` を尊重するようになった
・PowerShell ツールのコマンドが Bash と同様にパーミッションモードで自動承認されるようになった
・Hooks: `PostToolUse` および `PostToolUseFailure` フックの入力に `duration_ms`（ツール実行時間）が含まれるようになった
・サブエージェントと SDK の MCP サーバー再設定が直列ではなく並列で接続するようになった
・別プラグインのバージョン制約でピン留めされたプラグインが条件を満たす最新の git タグに自動更新されるようになった
・Vim モード: INSERT モードで Esc を押してもキューに入ったメッセージが入力欄に戻らなくなった。再度 Esc を押すと中断できる
・スラッシュコマンドの候補に、クエリに一致した文字がハイライト表示されるようになった
・スラッシュコマンドピッカーで説明が長い場合、切り捨てではなく 2 行目に折り返されるようになった
・出力の `owner/repo#N` 短縮リンクが常に github.com ではなく git リモートのホストを使用するようになった
・セキュリティ: `blockedMarketplaces` の `hostPattern` および `pathPattern` エントリが正しく適用されるようになった
・OpenTelemetry: `tool_result` および `tool_decision` イベントに `tool_use_id` が追加。`tool_result` には `tool_input_size_bytes` も含まれる
・ステータスライン: stdin JSON に `effort.level` と `thinking.enabled` が含まれるようになった
・Vertex AI でのツール検索がデフォルトで無効になった（`ENABLE_TOOL_SEARCH` でオプトイン可能）
・CRLF コンテンツ（Windows クリップボード、Xcode コンソール）の貼り付けで各行間に余分な空行が挿入されていた問題を修正
・kitty キーボードプロトコルを使用するターミナルで複数行貼り付け時に改行が失われていた問題を修正
・パーミッションで Bash ツールが拒否された際に macOS/Linux ネイティブビルドで Glob および Grep ツールが消える問題を修正
・フルスクリーンモードでスクロールアップしてもツール完了のたびに最下部に戻る問題を修正
・OAuth ディスカバリーリクエストに対してサーバーが非 JSON ボディを返した場合に MCP HTTP 接続が失敗していた問題を修正
・画像添付ファイルを含むメッセージで Rewind オーバーレイに "(no prompt)" が表示されていた問題を修正
・自動モードがプランモードを "Execute immediately" 指示で上書きしていた問題を修正
・レスポンスペイロードを送出しない非同期 `PostToolUse` フックがセッション記録に空のエントリを書き込んでいた問題を修正
・サブエージェントのタスク通知がキューで孤立した場合にスピナーが表示され続けていた問題を修正
・絶対パスを使ったスラッシュコマンド内で `@` ファイルの Tab 補完を使用するとプロンプト全体が置き換えられていた問題を修正
・Docker や SSH 経由で macOS Terminal.app を使用した場合、起動時にプロンプトに不要な `p` 文字が表示されていた問題を修正
・HTTP/SSE/WebSocket MCP サーバーの `headers` 内の `${ENV_VAR}` プレースホルダーがリクエスト前に置換されていなかった問題を修正
・`client_secret_post` を要求するサーバーのトークン交換時に `--client-secret` で保存された MCP OAuth クライアントシークレットが送信されていなかった問題を修正
・`/skills` ダイアログで Enter キーを押すとダイアログが閉じるだけで、プロンプトに `/<skill-name>` が入力されなかった問題を修正
・`/agents` 詳細ビューでサブエージェントが利用できない組み込みツールが "Unrecognized" と誤ってラベル付けされていた問題を修正
・プラグインキャッシュが不完全な場合に Windows でプラグインからの MCP サーバーが起動しなかった問題を修正
・`/export` が実際に使用されたモデルではなく現在のデフォルトモデルを表示していた問題を修正
・verbose 出力設定が再起動後に保持されなかった問題を修正
・`/usage` のプログレスバーが "Resets …" ラベルと重なっていた問題を修正
・`${user_config.*}` が空のオプションフィールドを参照した場合にプラグイン MCP サーバーが失敗していた問題を修正
・文末に数字を含むリストアイテムで数字が独立した行に折り返されていた問題を修正
・プランモードに入る際に `/plan` および `/plan open` が既存のプランに作用しなかった問題を修正
・自動コンパクション前に呼び出されたスキルが次のユーザーメッセージに対して再実行されていた問題を修正
・`/reload-plugins` および `/doctor` が無効化されたプラグインのロードエラーを報告していた問題を修正
・`isolation: "worktree"` を使用した Agent ツールが以前のセッションの古い worktree を再利用していた問題を修正
・無効化された MCP サーバーが `/status` で "failed" と表示されていた問題を修正
・`TaskList` がタスクを ID でソートせず任意のファイルシステム順で返していた問題を修正
・`gh` の出力に "rate limit" を含む PR タイトルがある場合に誤った "GitHub API rate limit exceeded" ヒントが表示されていた問題を修正
・SDK/bridge の `read_file` が増大するファイルのサイズ上限を正しく適用していなかった問題を修正
・git worktree で作業する際に PR がセッションにリンクされていなかった問題を修正
・`/doctor` が高優先順位のスコープで上書きされた MCP サーバーエントリについて警告していた問題を修正
・Windows: 誤検知の "Windows requires 'cmd /c' wrapper" MCP 設定警告を削除
