🔧 **Claude Code アップデート**
**v2.1.121**
＜新機能・改善＞
・MCP サーバー設定に `alwaysLoad` オプションを追加 — `true` に設定すると、そのサーバーからのすべてのツールがツール検索の遅延をスキップして常に利用可能になる
・`claude plugin prune` コマンドを追加して孤立した自動インストールプラグイン依存関係を削除；`plugin uninstall --prune` でカスケード削除
・`/skills` に型で絞り込むための検索ボックスを追加し、長いリストを見やすくした
・PostToolUse フックが `hookSpecificOutput.updatedToolOutput` を使用して全ツールのツール出力を置換可能に（以前は MCP のみ）
・フルスクリーンモード：プロンプトに入力してもスクロールが自動で下に戻らなくなった
・ターミナルをはみ出すダイアログを矢印キー、PgUp/PgDn、Home/End、マウスホイールでスクロール可能に（フルスクリーン・非フルスクリーン両対応）
・複数行に折り返った長い URL のいずれかをクリックすると、フルスクリーンモードで完全な URL が開く
・SDK と `claude -p`：`CLAUDE_CODE_FORK_SUBAGENT=1` が非対話型セッションで動作するようになった
・`--dangerously-skip-permissions` が `.claude/skills/`、`.claude/agents/`、`.claude/commands/` への書き込みでプロンプトを表示しなくなった
・`/terminal-setup` が iTerm2 の「ターミナル内のアプリケーションがクリップボードにアクセスできる」設定を有効にし、tmux からも `/copy` が動作するように
・MCP サーバーが起動時に一時的なエラーに遭遇した場合、最大 3 回まで自動リトライ
・ターミナルタブセッションタイトルが設定された `language` で生成されるように
・同じ上流 URL を持つ Claude.ai コネクタが重複排除されるように
・Vertex AI：X.509 証明書ベースの Workload Identity Federation（mTLS ADC）をサポート
・アップグレード後の起動が高速化：リリースノートスプラッシュから Recent Activity パネルを削除
・LSP 診断概要がクリック/Ctrl+O で展開可能に、展開ヒントを表示
・SDK：`mcp_authenticate` が `redirectUri` をサポート（カスタムスキーム完了および claude.ai コネクタ用）
・OpenTelemetry：LLM リクエストスパンに `stop_reason`、`gen_ai.response.finish_reasons`、`user_system_prompt`（`OTEL_LOG_USER_PROMPTS` でゲート）を追加
・[VSCode] 音声ディクテーションが Claude Code 言語が設定されていない場合、`accessibility.voice.speechLanguage` 設定を尊重するように
・[VSCode] `/context` でネイティブトークン使用量ダイアログが開くように
＜バグ修正＞
・セッション内で多くの画像を処理するときの無制限メモリ増加（複数 GB RSS）を修正
・`/usage` が大規模なトランスクリプト履歴のマシンで最大 〜2GB のメモリをリークしていたのを修正
・長時間実行ツールが明確な進行状況イベントを発行しないときのメモリリークを修正
・Claude が開始したディレクトリが削除または移動されたときに Bash ツールが永久に使用不可になるのを修正
・`--resume` が外部ビルドで起動時にクラッシュするのを修正
・`--resume` がトランスクリプト行が非クリーンなシャットダウンで破損した場合の大規模セッションで失敗するのを修正（破損した行はスキップされるように）
・Bedrock application inference profile ARN 使用時に「thinking.type.enabled is not supported」エラーが出るのを修正
・Microsoft 365 MCP OAuth が重複またはサポートされていない `prompt` パラメータで失敗するのを修正
・`Ctrl+L` トリガーや再描画時の非フルスクリーンモード tmux、GNOME Terminal、Windows Terminal、Konsole のスクロールバック重複を修正
・claude.ai MCP コネクタが起動時にコネクタリスト取得が一時的な認証エラーに遭遇したときに通知なく消えるのを修正
・リモートセッションで組み込みツールの「常に許可」ルールがワーカー再起動後も持続しないのを修正
・ネイティブビルド下で `managed-settings.json` 経由で設定した場合、`NO_PROXY` がすべての HTTP クライアントで尊重されないのを修正
・マネージド設定承認プロンプトが承認時にセッションを終了するのを修正（設定を適用して継続するように）
・`/usage` が古い OAuth トークンの後に「rate limited」を返すのを修正（自動更新されるように）
・`settings.json` の無効なレガシー列挙値がファイル全体を無効にするのを修正
・`/usage` ダイアログコンテンツが no-flicker モードオフで切り取られるのを修正
・フルスクリーンレンダラーがオフのときに `/focus` が「Unknown command」を表示するのを修正（有効化方法を説明するように）
・セッション中にバイナリが削除されたときに組み込まれた grep/find/rg シェルラッパーが失敗するのを修正（インストール済みツールへフォールバック）
・大規模なディレクトリツリーで Bash ツールの `find` 中のピークファイルディスクリプタ使用量を削減

**v2.1.120**
＜新機能・改善＞
・Windows：Git for Windows（Git Bash）は不要に — 存在しない場合、Claude Code は PowerShell をシェルツールとして使用
・`claude ultrareview [target]` サブコマンドを追加して `/ultrareview` を CI またはスクリプトから非対話的に実行（stdout に結果を出力、`--json` で生データ出力、完了時は 0、失敗時は 1 で終了）
・スキルが `${CLAUDE_EFFORT}` で現在のエフォートレベルを参照できるように
・サブプロセスに `AI_AGENT` 環境変数を設定して `gh` が Claude Code トラフィックを属性づけできるように
・デスクトップアプリケーション作成またはスキル/エージェント作成を推奨するスピナーティップスを、既に持っている場合は非表示に
・ターミナルが矢印キーの代わりにスクロールイベントを送信する場合は「PgUp/PgDn をスクロールに使用」ヒントを表示
・claude.ai コネクタが多数設定されているが認可されていない場合のセッション開始が高速化
・auto モード拒否メッセージが設定ドキュメントにリンク
・`claude plugin validate` が `marketplace.json` トップレベルで `$schema`、`version`、`description`、`plugin.json` で `$schema` を受け入れるように
・auto モード内自動コンパクトが誤解を招く全トークン値の代わりに `auto`（小文字、トークンカウントなし）を表示
＜バグ修正＞
・`/rewind` および他の対話型オーバーレイが `claude --resume` で起動後にキーボード入力に応答しないのを修正
・非フルスクリーンモード（リサイズ、ダイアログ解除、長時間セッション）でターミナルスクロールバック重複を修正
・`DISABLE_TELEMETRY` / `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` が API および enterprise ユーザーの使用メトリクステレメトリを抑制しないのを修正
・auto モードで複数行 bash コマンドにパイプとリダイレクト両方が含まれる場合の誤検知「危険な rm 操作」許可プロンプトを修正
・フルスクリーンモードで長い選択メニューがターミナルの下に切り取られるのを修正（フォーカスオプションがスクロール時に画面上に留まるように）
・フルスクリーンで Write ツール出力が「+N lines」をクリックしたときに折りたたまれるのを修正（展開されるように）
・スラッシュコマンドピッカーが入力中にジャンプし、ハイライトが青でのみ連続部分文字列にマッチするのを修正
・`/plugin` マーケットプレイスが 1 つのエントリが認識されていないソース形式を使用している場合にロード失敗するのを修正
・[VSCode] `/usage` がネイティブアカウント＆使用量ダイアログを開くように（プレーンテキスト返却の代わりに）
・[VSCode] 音声ディクテーションが `~/.claude/settings.json` の `language` 設定を尊重するように
・大規模なディレクトリツリーで Bash ツールの `find` がオープンファイルディスクリプタを枯渇させ、ホスト全体のクラッシュを引き起こすのを修正（macOS/Linux ネイティブビルド）
